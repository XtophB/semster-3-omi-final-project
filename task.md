* Mitarbeiter:innen des Pizzalieferdienstes „Amore Mio“ sind entweder Pizzaköch:innen, Küchenhilfen oder Lieferfahrer:innen. Außerdem gibt es noch eine Person, welche den gesamten Ablauf koordiniert. Pizzaköch:innen, Küchenhilfen und Koordinator:innen erhalten einen fixen Lohn pro Stunden. Lieferfahrer:innen erhalten ebenfalls einen fixen Lohn pro Stunde sowie einen Extrabonus für ausgelieferte Bestellungen. Für jede Mitarbeiter:in sollte es jedenfalls letztendlich einen Mechanismus (z.B. eine Methode) zur Berechnung des Lohns geben.
* Eine Pizza besteht aus diversen Zutaten (Teig, Tomatensauce und die verschiedenen Arten von Zutaten für den Belag). Diese Zubereitungsart kann jedoch variieren. So kann der Lieferdienst z.B. an gewissen Tagen in der Woche auf die typisch italienische (bzw. neapolitanische) Pizza (dünner Teig, aus dem Holzofen, dünn belegt) spezialisiert sein. Es kann jederzeit sein (z.B. an einem “Special American Pizza Day”), dass auf Pizzen nach eher amerikanischer Art (dicker Teig, aus dem Elektroofen, reichhaltiger belegt) umgestellt wird. Nicht zu vergessen sind dabei auch nicht die verschiedenen Pizzenvarianten aus dem süd-ost europäischen Raum, auf die auch jederzeit umgestellt werden kann, bzw. es spezielle Aktionstage geben kann. Nachdem sich also Geschmäcker bzw. Vertriebskonzepte jederzeit ändern können, sollten die Auswahl an Pizzen sowie die Zubereitungsart (bzw. die dazugehörige Erstellung von „Pizzaklassen“ und deren Instanzen) jedenfalls leicht austauschbar sein. Dabei ist auch noch zu berücksichtigen, dass es in Zukunft ja auch neue Filialen mit individueller Auswahl an Pizzen und entsprechender Zubereitungsart geben kann. Das Konzept von Filialen sollte also von vornherein mitgedacht bzw. miteinbezogen werden.
* Der Lieferdienst bzw. künftige Filialen brauchen auch eine Lagerverwaltung für die verschiedenen Zutaten. Dabei sind Zutatentyp, gewünschte Lagermenge, tatsächliche Lagermenge und das Verfallsdatum festzuhalten. Werden Zutaten verbraucht, so soll sich das auch auf ihre tatsächliche Lagermenge entsprechend auswirken. Die Befüllung des Lagers ist nicht Teil dieses Systems und wird von externen Lieferant:innen übernommen.
* Kund:innen können ihre Pizza noch selbst dynamisch anpassen, und zusätzliche Toppings hinzufügen (Käse (Parmesan, Mozarella, Feta), Schinken (Crudo und Cotto), Fleisch (Hühnchen, Carpaccio), Champignons, Jalapenos, Salami, Zwiebel, Mais und Chilli-Flocken. Wenn Kund:innen Toppings hinzufügen, soll der Preis der
Pizza bestimmt werden, indem die Preisbestimmung auf Implementierungsebene delegiert wird. 
* Die Bestellung, Zubereitung und Auslieferung von Pizzen laufen grundsätzlich so ab:
  * Kund:innen erstellen Bestellungen mit einer oder mehreren Pizzen, welche über ein Online-App getätigt werden. Bei Bezahlung der Bestellung kann man aus zwei verschieden Arten wählen: Kreditkarte und Zahlung in bar bei der Lieferung. Trifft eine Bestellung ein, so wird diese von der jeweiligen koordinierenden Person kurz geprüft. Bei Bedarf erfolgt eine Rückfrage an die jeweilige Kund:in. Es kann also z.B. sein, dass eine Bestellung leider überhaupt nicht bearbeitet werden kann, diverse Zutaten (und Toppings) fehlen oder dass die Bestellung nur mit großer Verzögerung eintreffen würde, da der Betrieb überlastet ist.
  * Ist alles okay, so erhält die Kund:in eine Bestellbestätigung und die Bestellung wird an die anderen Mitarbeiter:innen weitergegeben. Idealerweise werden Bestellungen jeweils freien Mitarbeiter:innen bzw. je
nach deren Auslastung zugeteilt. Die Bestellungen werden grundsätzlich je nach Eingangszeit nacheinander abgearbeitet. 
  * Eine Küchenhilfe übernimmt dann die Vorbereitung der Bestellung (Teig,Tomatensoße, Belag). Eine Pizzaköch:in bäckt danach die Pizza bzw. die Pizzen der Bestellung. Wird dabei eine Pizza verbrannt, so muss sie natürlich von Beginn an neu zubereitet werden.
  * Sobald alle Pizzen einer Bestellung fertig gebacken sind, übernimmt eine Lieferant:in die Bestellung und liefert diese aus. Beim Zeitpunkt der Übernahme durch eine Lieferant:in erhält die jeweilige Kund:in eine weitere Nachricht mit der Information, dass die Bestellung unterwegs ist und die geschätzte Lieferdauer. Kund:innen sollten nicht dauernd manuell über die Bestell-App abfragen müssen, wie es um ihre Bestellung steht, sondern sie werden entsprechend benachrichtigt.
* Im Zuge der Umsetzung werden voraussichtlich auch Klassen benötigt werden, die andere Klassen bzw. deren Objekte verwalten (z.B. eine Mitarbeiter:innen – Verwaltung). Von diesen Verwaltungsklassen solle es nach Möglichkeit jeweils nur eine Instanz im System geben.
